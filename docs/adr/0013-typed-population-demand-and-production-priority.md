# Typed population demand and production priority

Actionable work creates demand for the Nanobot Type that can perform it, while Production Priority orders competing shortages rather than defining a target population mix. Existing and in-production capacity counts only for its own type, zero priority cannot suppress required work, and excess nanobots are retained when demand falls. This prevents surplus capacity in one role from hiding an unmet role while avoiding automatic retirement or unbounded composition-correction production.
