# World-space nanobots

Nanobots are top-level entities with world-space `Transform`s, not children of a positioned swarm entity. Originally bots were spawned via `with_children` under the swarm, making their `Transform.translation` local to the swarm parent; but every movement, gather, build, and arrival system reads `transform.translation` as a world coordinate and steers toward world-space destinations. With the player swarm parent at a half-cell offset `(256, 256)`, bots navigated until their *local* translation equalled the *world* destination and ended up offset by `(+256, +256)` — the top-right corner of every target cell. The opponent swarm was broken by the same offset. Tests missed it because `tests/common/mod.rs` spawns bots top-level (local == world), so only `cargo run` exhibited it. Making nanobots top-level entities — carrying `SwarmMember`/`OwnerSwarm` for ownership, with the swarm entity reduced to a pure ownership/spawn-origin marker — makes the existing world-space code correct as written and removes the whole class of local/world bugs. Rejected alternative: keep bots parented and switch every system to `GlobalTransform`; that preserves a parenting relationship carrying no semantic weight (the glossary's `Swarm` has no fixed subgroups, and nothing ever moves the swarm `Transform` after spawn) while keeping the same trap loaded.
