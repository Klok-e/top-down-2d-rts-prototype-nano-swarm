# Tiered logistics flow with terminal consumers

The material chain has three directed hauler legs — deposit (worker) → source stockpile (hauler) → sink stockpile (hauler) → terminal — and haulers pick legs sink-first, ranked downstream-first (terminal before buffer), nearest-within-tier breaking ties. Terminals (production facilities and chargers) own their own input buffer as a struct field, like `Charger` does today; they are never stamped with `Stockpile` + `StockpileRole::Sink` and never enter stockpile queries, so the buffer production consumes is exactly the buffer haulers fill and stockpile-flow can't bypass leg 3. A sink stockpile is a bidirectional node: sink in leg 2, source in leg 3, so `StockpileRole` marks a stockpile's place in the chain rather than a fixed flow direction.

Facilities and chargers are treated asymmetrically. The `sink_stockpile_demand_system` auto-plans one sink stockpile per facility, searching the painted Build Zone (not just the facility's own cell) for non-overlapping space so a packed cell doesn't starve the facility; chargers are direct-delivery terminals fed straight by haulers, because they live in Defend zones with no Build paint and CONTEXT.md themes their logistically-isolated degradation as intended player pressure.
